Neotoc Playground

Uses Neotoc v(NEOTOC_VERSION)

Thrilled to have you here! Play around with the files below, especially those in src, index.html, and dummy-content.conf, and have fun experimenting with the commands!

File structure:

.
├── index.html ---------> This is what you are viewing...
├── package.json
├── dummy-content.conf -> Allows you to easily configure
│                         the dummy content below.
├── src
│   ├── base.css -------> General styles for this page.
│   ├── main.js --------> Neotoc is called here ✨
│   └── neotoc.css -----> Ready-made Neotoc styles are
│                         customized here.
├── public
│   └── favicon.ico
├── .prettierignore
├── .prettierrc
└── .gitignore

package.json scripts available for your use:

Command Action
(packageManager) run fill Shuffle the dummy content below.
(packageManager) run fill-prettily Same as above but prettify the HTML output.
(packageManager) run dev Run the dev server.
(packageManager) run build Build this project.
(packageManager) run preview Preview the build.

Vite is used here as the dev server and build tool.

The dummy-content.conf file serves as the configuration file for the dummy-content CLI, which is used to generate random Lorem Ipsum content with hierarchical heading levels as shown below.

If you don't want any depth in the heading hierarchy, set deepness to 0 in this file. To disable numbering in the headings, set no-numbering to true. Remember to run (packageManager) run fill afterward to regenerate dummy content and replace the existing content in the element with the ID content.

To explore more ways to customize the dummy content, run the following command in your terminal to view its help documentation. 

npx dummy-content --help
